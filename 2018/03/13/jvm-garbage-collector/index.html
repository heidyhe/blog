Garbage Collection in JVM

Posted on 2018-03-13 | In java

Unfinished.

Garbage collection frees the memory (in heap space) occupied by the objects that are no longer in use. It takes this job from programmers who often forget to release the memory when they no longer need it and sometimes find it hard to put the piece of code in the right place.
What JVM benefits from garbage collection other than preventing programmers from messing up with the memory? The answer is more efficient at allocating memory for new objects. This is because garbage collection not only frees memory but also compacts objects in the heap. I wonder if other languages without garbage collector have some similar mechanisms for compaction.

This post is about how garbage collection is implemented in JVM. In Part I, I’ll walk through some algorithms of garbage collection which are discussed in Thinking in Java. In Part II, I’ll write about how garbage collection is implemented in JVM nowadays.

Part I. Garbage collection algorithms

Reference counting

There is a reference counter for each object. If a new reference is attached to the object, the reference counter is increased by 1; if a reference is detached from the object, the counter is decreased by 1.
